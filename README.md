## Tables

- 序号：从 1 开始的自然数
- 名称：ens 名称。【如果包装，高亮显示】
- 所有者：显示名称的所有者。【如果与连接的钱包相同，则高亮显示】
- 状态：显示名称的当前状态
- 元数据：显示 labelhash 和 namehash 的复制链接
- 相关信息：显示与这个名称有关的链接，暂时留空
- 操作：显示根据名称的状态显示注册或更新按钮 【如果钱包没连接，则为不可用状态】
- 删除：显示多选框用来单个或批量删除（只在 home 页面显示这一列，由于 CollectionDetail.tsx 中显示的是固定数量的名称，因此不需要这一列）

若某列筛选生效，则这一列的表头高亮显示。

## Prompts

让我们回到表头功能的开发，以下是每个表头内需要添加的功能：

- 序号：暂时不需要。
- 名称：（1）按照标签字母排序，（2）按长度排序。默认按照获取信息结果的默认顺序
- 所有者：（1）就像你之前提到的，可以选择只显示当前账户拥有的名称。
- 状态：（1）按照状态排序，即按照过期时间排序，（2）按照状态分类筛选，使用多项选择下拉菜单。
- 元数据：暂时不需要。
- 相关信息：暂时不需要。
- 操作：（1）按照可注册、可更新筛选，选项：全部/注册/更新，默认显示全部。
