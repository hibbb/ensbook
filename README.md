- 所有者：显示名称的所有者。【如果与连接的钱包相同，则高亮显示】
- 状态：显示名称的当前状态。
- 相关信息：显示与这个名称有关的链接，暂时留空
- 操作：显示根据名称的状态显示注册或续费按钮 【如果钱包没连接，则为不可用状态】

## Prompts

actioncell 中的 主操作按钮，采用了样式和锚文本分别进行判断并显示不同内容的逻辑，但这种方式在实际操作中已经发现了样式和文本错配的情况，请选择一种更加稳健的方式来构建主操作按钮。
我提到的样式和文本错配是在这种情况下发生的：注册某个名称的程序启动后，调用 makeCommitment 生成了commitment  并保存在本地，由于某种原因，用户在此中断了操作，并在其他浏览器中完成了注册操作，当用户返回到本页面时，程序在本地检索到这个名称的注册记录，并判定状态为 isPending = true，因此样式为断点续传的按钮样式，但由于这个名称实际上已经完成了注册，按钮的锚文本显示的是“续费”，最终发生了我说的文本和样式错配。
从这个错配来说，我们不仅要改进主操作按钮的显示逻辑，还要改进注册进程保存、读取和删除逻辑，比如当用户只是生成了 commitment 却没有提交上链，这种情况下，还有没有必要进行记录，以及其他你可能想到的逻辑漏洞。

表头的序号格内目前显示了一个 # 号，我希望将其修改为显示数字，显示逻辑如下：
当用户没有进行任何筛选时，显示当前表格中的名称数量，也就是总行数。
当用户进行筛选之后，将筛选后的行数与总行数进行组合显示。
